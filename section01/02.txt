
리액트의 장점 : 빠른 업데이트 and 렌더링 속도
--> 업데이트 : 웹사이트를 탐색할 때, 화면에 나타나느 내용이 바뀌는 것

리액트는 빠른 업데이트를 위해 Virtual DOM 이란 것을 사용한다 (가상 돔)

DOM : Document Object Model --> 웹페이지를 정의하는 하나의 객체 (하나의 웹사이트에 대한 정보를 모두 담고 있는 큰 그릇)

Virtual DOM 은 어떻게 업데이트를 빠르게 하는가? :
    일반적으로 사용자와 상호작용을 하는 웹페이지의 경우 화면의 업데이트가 수시로 이뤄진다 --> 곧 DOM 이 수정된다는 말과 같다
    기존의 방식으로 화면을 업데이트 하려면 DOM 을 직접 수정해야하는데, 이것은 성능에 영향을 크게 미치고, 비용도 많이 잡아먹는다.

    반면에 React는 DOM을 직접 수정하는 것이 아니라 업데이트 해야할 최소한의 부분만 찾아서 업데이트를 한다
    업데이트해야할 최소한의 부분만 검색 -> 검색된 부분만을 업데이트 -> 다시 랜더링하면서 변경된 내용을 빠르게 사용자에게 보여줄 수 있다


리액트의 장점 : Component 구조 (레고 블록 조립하듯 컴포넌트들을 모아서 개발)


리액트의 장점 : 재사용성 (개발 기간이 단축된다, 유지 보수가 용이하다)
